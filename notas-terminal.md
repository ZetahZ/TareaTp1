# Notas de Comandos de Terminal y Git - TP1

En este proyecto utilicé la terminal para gestionar archivos y el sistema de control de versiones. Estos son 10 comandos usados en este tp:

1. **pwd**: Lo usé para confirmar que estaba en la carpeta correcta del TP antes de empezar.
2. **mkdir**: Para crear la carpeta del proyecto desde cero.
3. **touch**: Para crear los archivos index.html, style.css y los documentos .md.
4. **cd**: Para entrar y moverme entre las carpetas de mi proyecto.
5. **ls**: Para listar los archivos y verificar que todo se haya creado bien.
6. **git init**: El comando más importante de Git. Lo usé para inicializar el repositorio en mi carpeta local.
7. **git config --global user.name "TuNombre"**: Para identificarme como el autor de los cambios en el proyecto.
8. **git config --global user.email "tu@email.com"**: Para vincular mi correo a las versiones guardadas en Git.
9. **git status**: Lo usé todo el tiempo para ver qué archivos estaban marcados para guardarse (en verde) o cuáles eran nuevos (en rojo).
10. **code .**: Para abrir rápidamente todo el proyecto en Visual Studio Code.
11. **cat**: Lo usé para leer el contenido de un archivo directamente en la terminal sin tener que abrir vs code.
12. **which**: Sirve para localizar la ruta de un comando o programa (por ejemplo, `which git`). Sirve por ejemplo para verificar que Git estuviera bien instalado. Al usar el comando which git para entontrar la direccion de git, la consola tiro la siguiente direccion: /mingw64/bin/git

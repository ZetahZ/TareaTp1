# Reflexión sobre Metodologías Ágiles - TP1

### (a) ¿Qué diferencia hay entre Scrum y Kanban? ¿Cuál te parece más útil para un proyecto individual y por qué?
La diferencia principal es que Scrum se basa en ciclos de tiempo fijos llamados Sprints con roles definidos, mientras que Kanban es un flujo continuo visualizado en un tablero. Para un proyecto individual como este sitio web, me parece más útil Kanban porque permite gestionar las tareas de HTML, CSS y Git de forma flexible y sin la presión de tiempos rígidos. Me ayudó a ver el progreso real de cada sección del código sin necesidad de ceremonias complejas.

### (b) ¿Qué es un MVP (Producto Mínimo Viable) y cómo lo aplicarías si te piden hacer un sitio web para un negocio local?
Un MVP es la versión más básica y funcional de un producto que permite obtener feedback de los usuarios con el menor esfuerzo de desarrollo posible. Si tuviera que hacer un sitio para un negocio local, mi MVP sería una página con la información de contacto, fotos de los productos y un mapa de ubicación. No agregaría funciones complejas como pagos online o sistemas de usuarios hasta confirmar que la base del sitio funciona y es útil para los clientes.

### (c) ¿Cómo te ayudó el tablero Kanban a organizar las tareas de este TP? Mencioná un ejemplo concreto.
El tablero Kanban fue fundamental para mantener el orden entre los requisitos técnicos de accesibilidad y el manejo de versiones con Git. Un ejemplo concreto fue cuando separé la tarea de "Maquetación del Formulario" de la de "Validación Visual con CSS". Ver cómo las tarjetas pasaban de la columna 'En proceso' a 'Hecho' me permitió asegurar que no me olvidara de ningún punto de la consigna antes de hacer el merge final.
